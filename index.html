My Notes from "The Basics of the Web and HTML"

The Fundamentals

 The world wide web is, fundamentally, a collection of approximately 30 billion individual web pages and has been around since the 1990s. Each of these pages is written in HTML (discussed in more detail below) and is connected to other pages by links. To navigate the web, a user's browser on their computer sends an HTTP request to a server, which then replies with the requested HTML document. It is the user's browser's job to read the HTML document and translate it into the site/content that is ultimately displayed on the user's screen. This Udacity video provides a nice overview of the process.

HTML

 HTML, or Hypertext Markup Language, is the language in which most of the content on the internet is written. There are a few components: text content (what we see), markup (what it looks like), references to other documents (such as images and videos), and links to other pages. See this Udacity video for more details.

Coding in HTML

 Each HTML page is made up of any of a number of individual elements. These elements are defined using tags; as with elements, there is a huge variety in tags. Examples include the 'strong,' 'b,' and 'br.' The first two make text elements bold and the the third breaks the current paragraph and starts a new one. Once again, Udacity has a video outlining these concepts.

A Note on HTML Elements

 One important distinction between HTML elements is whether or not they are block or inline. Block elements are easiest to understand if you picture an invisible box or block being drawn around the content enclosed by a block tag. Block elements represent easy ways to help put structure around a page's text. The paragraph structure of this very page was built using the 'p,' or paragraph tag. Inline tags such as 'strong,' or 'em' have no impact on the organizational structure of the text, just the way it looks.

Non-Code Takeaways

 First, computers are stupid. It isn't a derogative term--it simply highlights the fact that computers do exactly what a user or programmer tells them to do. There is 0 room for creative interpretation by computers. While this makes their behavior predictable, it also leaves room for minor typos to have major impacts on the behavior of a program or website being coded.

Second, and this is something I really appreciate, it's okay to forget the details. It's important to understand the underlying concepts and theory but Google (or Stack Overflow) are great resources for remembering the details of an individual tag or re-learning how to make something behave the way you want it to. There's simply too much for any one person to remember.
